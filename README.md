# AMP — Advertiser Management Platform

A unified multi-DSP campaign management platform by Alkimi Exchange.

## Overview

AMP (Advertiser Management Platform) consolidates multi-DSP advertising operations into a single, streamlined workflow. Instead of managing campaigns separately across DV360, Beeswax, and Alkimi, teams can create, launch, and report on campaigns from one unified interface.

**Live Demo:** [View Landing Page](https://yourusername.github.io/amp-landing-page/)

---

## Key Features

| Feature | Description |
|---------|-------------|
| **Unified Campaign Management** | Create once, deploy across all DSPs |
| **IO Ingestion Automation** | Auto-extract budgets, dates, and KPIs from insertion orders |
| **Cross-DSP Reporting** | Normalized metrics with AI-powered insights |
| **Unified Audience Sync** | Select audiences once, sync to every platform |
| **Web3-Native Targeting** | Built-in support for decentralized advertising |
| **Role-Based Access** | Tailored views for AdOps, Account Managers, and Marketers |

---

## Target Users

- **AdOps Specialists** — Reduce repetitive campaign setup across platforms
- **Account Managers** — Access real-time dashboards and generate client-ready reports
- **In-House Marketers** — Automate execution to focus on strategy

## Local Development

To run this project locally:

```bash
# Clone the repository
git clone https://github.com/yourusername/amp-landing-page.git

# Navigate to the project folder
cd amp-landing-page

# Open in browser (macOS)
open index.html

# Or start a local server (Python 3)
python -m http.server 8000
# Then visit http://localhost:8000
```

## Deployment

This site is deployed using GitHub Pages.

**To deploy your own copy:**

1. Fork this repository
2. Go to **Settings** → **Pages**
3. Set source to `main` branch
4. Your site will be live at `https://yourusername.github.io/amp-landing-page/`
